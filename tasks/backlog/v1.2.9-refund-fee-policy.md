# INT3.3 — Refund Fee Policy Decision

## Source
v1.2.9 external audit finding INT3.3 (P1)

## Current Behavior
When a payment intent is refunded, the gateway fee (2% default) IS refunded
along with the principal amount. For a 15.0 credit payment with 2% fee:
- Payer charged: 15.0 + 0.30 = 15.30
- On refund: payer receives 15.30 back (principal + fee)

## Audit Expectation
The audit expected only 15.0 to be refunded (fee retained by platform).

## Decision Needed
**Should the gateway fee be non-refundable on refund?**

Options:
1. **Keep current behavior** (full refund including fee) — simpler, more
   consumer-friendly, standard in most payment platforms.
2. **Retain fee on refund** — more revenue for platform, but may frustrate
   agents and complicate accounting.
3. **Configurable per-tier** — free tier retains fee, pro tier refunds fee.

## No Code Change
This is a product decision. No code change in the current PR.
Defer to product owner.
