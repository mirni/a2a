## HIGH Priority

### H1: Gatekeeper Z3 100% Failure
**Status:** Fix in progress (PR #100 — `fix/z3-gatekeeper-deploy`)
**Root cause:** Two bugs:
1. **Packaging:** boto3 not installed on host, `VERIFIER_AUTH_MODE` / `VERIFIER_LAMBDA_FUNCTION` / `VERIFIER_LAMBDA_REGION` not set in `.env`. PR #100 fixes postinst scripts + CI smoke test.
2. **Runtime:** `VerifierClient.__init__()` succeeds (lazy boto3 import), but every `invoke()` fails with `ModuleNotFoundError`. Fixed by adding eager boto3 probe in `lifespan.py` — falls back to mock if probe fails.

**Fix:** Merge PR #100. The boto3 probe is included in this PR.
**Verification:** After deploy, `POST /v1/gatekeeper/jobs` with a trivial SAT expression should return `status=completed, result=satisfied`.

### H2: PyPI SDK v1.2.1 (11 versions behind)

**Note:** Trusted publishing via OIDC is already configured in `release.yml` workflow. Tag `sdk-v1.4.0` should auto-publish.

### H3: npm SDK Parameter Shapes Broken
**Owner:** Human (requires npm publish token)

## MEDIUM Priority

### M1: No `/metrics` Endpoint (Prometheus)
**Persistent for:** 8+ releases
**Fix:** Wire `PrometheusMiddleware` from `starlette_exporter` or `prometheus_fastapi_instrumentator` into `app.py`. Expose `GET /metrics` behind admin auth (or internal network only).
**Implementation:**
```python
# gateway/src/app.py
from prometheus_fastapi_instrumentator import Instrumentator
instrumentator = Instrumentator()
instrumentator.instrument(app).expose(app, endpoint="/metrics")
```
**Effort:** 0.5d including tests.

### M2: SSE Heartbeats Broken
**Regressed in:** v1.2.4
**Root cause:** `asyncio.sleep()` heartbeat loop in SSE handler was removed or broken during refactor. The `heartbeat_interval_seconds` config exists but the loop that sends `:heartbeat\n\n` comments is not executing.
**Fix:** In `gateway/src/routes/sse.py`, verify the heartbeat `asyncio.Task` is created and not immediately cancelled. Add a test that opens an SSE connection and asserts at least one heartbeat arrives within `heartbeat_interval_seconds * 2`.
**Effort:** 0.5d.

### M3: Messaging + Org Creation 500
**Persistent for:** 7+ releases
**Root cause:** Needs investigation. Likely a missing migration or schema mismatch in messaging or identity storage.
**Fix:** Reproduce with `POST /v1/messaging/channels` and `POST /v1/identity/orgs`, capture the 500 traceback, fix the underlying storage bug.
**Effort:** 1d (includes investigation).

### M4: HTTP Plaintext Not Redirected
**Owner:** Human (Cloudflare / infra config)
**Fix:** In Cloudflare dashboard → SSL/TLS → Edge Certificates → "Always Use HTTPS" = On. Or add a Page Rule: `http://api.greenhelix.net/*` → 301 to `https://api.greenhelix.net/*`.
**Note:** Gateway-side `FORCE_HTTPS=1` middleware already sends `Strict-Transport-Security` on HTTPS responses, but can't redirect plaintext if Cloudflare passes it through.

### M5: No Rate-Limit Headers
**Fix:** The `PublicRateLimiter` in `gateway/src/rate_limit_headers.py` already tracks limits. Add `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset` headers to responses. Standard middleware pattern.
**Effort:** 0.25d.

## CRITICAL (Config-Only)

### C1: Sandbox Stripe `cs_live_*`
**Persistent since:** v0.9.6
**Owner:** Human (Stripe dashboard)
**Fix:** In Stripe dashboard, switch sandbox environment to use test-mode keys (`sk_test_*`). Update `.env` on sandbox host: `STRIPE_API_KEY=sk_test_...`. The `assert_stripe_key_matches_env()` check in lifespan will prevent a production key from being used in a non-production `A2A_ENV`.
**Risk:** Until fixed, the sandbox could process real charges if a `create_checkout` call succeeds with a live key.
