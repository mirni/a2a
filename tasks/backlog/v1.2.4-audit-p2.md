## MEDIUM

### P2-M1 — Budget cap in legacy `/v1/execute` path
**File:** `gateway/src/routes/execute.py:507-520`
**Issue:** When `A2A_LEGACY_EXECUTE=1` is set, connector tools
dispatched through the legacy path skip the budget-cap check that
`gateway/src/deps/tool_context.py:162` applies to the REST routes.
**Fix:** Call `check_budget_cap` alongside the existing balance
check, or reject cap-controlled tools from the legacy path with
410 Gone.
**Test:** Extend `gateway/tests/v1/test_budget_cap_enforcement.py`
to also call `/v1/execute` with `A2A_LEGACY_EXECUTE=1` and assert
402 at the cap boundary.

### P2-M2 — Idempotency placeholder timeout
**File:** `gateway/src/deps/idempotency.py:128-186`
**Issue:** A handler crash between INSERT (status 0) and UPDATE
leaves a stale in-flight row forever. A replay matches body_hash,
sees status 0, proceeds — risking double-execution if the original
handler partially completed.
**Fix:** Use `status_code = -1` as the in-flight sentinel plus a
30-second TTL; after the TTL a retry is allowed to overwrite.
**Test:** `test_idempotency_placeholder_timeout_allows_retry`:
INSERT a row with status=-1 and created_at=35s ago, replay, assert
the replay wins.

### P2-M3 — Explicit admin bypass on deposit limits
**File:** `gateway/src/routes/v1/billing.py:164-175`
**Issue:** Admin bypass happens implicitly via
`config.deposit_limits.get("admin") → None`. Fragile; a typo in
config could accidentally expose enterprise behavior.
**Fix:**
```python
if tc.agent_tier == ADMIN_TIER:
    return  # admin bypasses deposit limits by design
```
plus a comment referencing this task ID.
**Test:** `test_admin_scoped_key_bypasses_deposit_limits` — admin
key deposits 100,000,001 credits → 200, not 400.

### P2-M4 — Idempotency row cleanup
**File:** `gateway/src/deps/idempotency.py:56-68`,
`gateway/src/lifespan.py`
**Issue:** No cleanup job for expired rows — table grows unbounded.
**Fix:** Start a background coroutine in `lifespan.py` that runs
`DELETE FROM idempotency_cache WHERE created_at < ? - :ttl`
hourly. TTL = 24h (matches existing replay window).
**Test:** `test_idempotency_row_cleanup_removes_expired_rows`.

## LOW

### P2-L1 — Well-known rate limit backstop
**File:** `gateway/src/middleware.py::PublicRateLimitMiddleware`
**Issue:** `/.well-known/*` routes have no per-IP rate limit in the
application layer. Cloudflare in front of the gateway should catch
DoS, but there's no application backstop.
**Fix:** Add `/.well-known/*` to the `PublicRateLimitMiddleware`
allow-list with a generous limit (e.g. 60/min/IP), or document
that all unauthenticated paths are assumed rate-limited at the edge.

### P2-L2 — Runtime health-latency histogram + alert
**File:** `gateway/src/routes/health.py`,
`monitoring/prometheus-alerts.yml`
**Issue:** Health SLO is only enforced at test time, not at runtime.
**Fix:** Emit a `gateway_health_latency_seconds` histogram on every
`/v1/health` request and add a Prometheus alert for
`p99 > 1s` sustained over 5 minutes.

### P2-L3 — NTP-safe cap window
**File:** `gateway/src/deps/tool_context.py:234-262`
**Issue:** Daily/monthly cap windows use `time.time()`, vulnerable
to backward NTP steps.
**Fix:** Add a code comment documenting the NTP-disciplined-clock
assumption, or switch to DB-managed boundaries via
`CURRENT_TIMESTAMP` in SQL.
